Modele de reference osi

Dans le tableau ci-dessous, nous avons spécifié les protocoles utilisés et l`unité de données échangées par chaque couche du modèle OSI. Aussi, vous pouvez parrainez cette vidéo YouTube impressionnante pour obtenir une image claire du modèle OSI, qui est expliqué en prenant l`analogie avec le train et le système ferroviaire. Le modèle de référence OSI est un modèle conceptuel qui utilise sept «couches» pour identifier les différentes fonctions fournies par un réseau, et ces sept couches peuvent être utilisées pour comparer différents protocoles à l`aide d`un cadre commun. Chaque couche dans le modèle de référence OSI a une fonction très spécifique, et chaque couche dépend des autres couches afin que le modèle entier fonctionne correctement. Chaque couche communique uniquement avec les calques immédiatement au-dessus ou en dessous. S`il y a un problème à une couche, il est de la responsabilité de cette couche spécifique de fournir des commentaires aux couches qui l`entourent. Ni le modèle de référence OSI ni les protocoles OSI ne spécifient d`interfaces de programmation, autres que des spécifications de service délibérément abstraites. Les spécifications de protocole définissent précisément les interfaces entre les différents ordinateurs, mais les interfaces logicielles à l`intérieur des ordinateurs, appelées sockets réseau, sont spécifiques à l`implémentation. Comme discuté dans la section précédente, le modèle OSI est composé de sept couches avec la couche d`application, qui est le plus proche de l`utilisateur final, en haut, allant tout le chemin vers le bas à la couche physique, dans lequel le transfert de données réel se produit avec l`utilisation d`une transmission mediu m.

OSI signifie «interconnexion de systèmes ouverts». La définition exacte du modèle OSI a déjà été donnée ci-dessus. Dans les mots plus simples, pour être précis, le modèle OSI est un outil utilisé par les professionnels de l`informatique pour modéliser ou tracer réellement le flux réel de la façon dont les transferts de données dans les réseaux. Ainsi, fondamentalement, le modèle OSI est un modèle logique/représentation de la façon dont les systèmes de réseau sont censés envoyer des données (ou, communiquer) les uns aux autres. Le modèle démarre en bas avec la couche physique (couche 1) et se termine en haut avec la couche d`application (couche 7). Le concept le plus important derrière le modèle est que chaque couche exécute une fonction spécifique, fournit des services à la couche au-dessus, et utilise les services de la couche en dessous. Il existe une interface bien définie entre chaque couche, dans laquelle le flux d`informations est maintenu délibérément minime. Il faut se rappeler que le modèle OSI lui-même n`est pas une architecture de communication. Il spécifie simplement ce que chaque couche doit faire, pas comment cela doit être réalisé. Comme indiqué ci-dessus, les protocoles dans la même couche à chaque extrémité de la liaison de communication ne peuvent communiquer les uns avec les autres que indirectement, en utilisant les services des couches ci-dessous. Les différentes couches du modèle de référence OSI sont résumées ci-dessous: le modèle de référence OSI est extrêmement utile comme outil pour discuter de divers services de réseau. Par exemple, si nous devions examiner un service réseau simple comme l`impression d`un document sur une imprimante connectée localement, nous pourrions utiliser le modèle de référence OSI pour déterminer comment cette tâche simple était réalisée.

Nous pourrions également utiliser le modèle pour déterminer comment l`impression sur un réseau Novell a été effectuée, ou comment l`impression sur un réseau TCP/IP a été accomplie. Étant donné que ces trois exemples utilisent le même modèle, ils peuvent tous être comparés les uns aux autres, même s`ils utilisent tous des technologies extrêmement différentes pour atteindre le même objectif. OSI avait deux composants majeurs, un modèle abstrait de mise en réseau, appelé modèle de référence de base ou modèle à sept couches, et un ensemble de protocoles spécifiques. En haut de la pile de modèles de référence OSI de couches, nous trouvons la couche application qui est implémentée par les applications réseau.
